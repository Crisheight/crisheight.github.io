Pokedex

Gotta call 'em all!

This project is all about using the pokedex API to cause a chain reaction of function calls and replies that we can then use along with css to create cards.

The html is standard, handing off the actually heavy lifting to CSS and Javascript.

Starting with JS, the idea is to create an elementId that can be used in the future, and pkmn number to provide a count for the API. Using that number, we can create a function to fetch the corresponding numbers (in this case the function keeps adding +1 until it hits 150). We set this function up as await, wherein the next function fetches the information (the number given) and gives us back the corresponding pokemon's data. We can set this up as creating a 'card'. We then use this 'card' to create another function that gives us more data and a visual. Things like what type it is, the name, and the sprite, are all pulled for our use. We inject some html here to actually let us see this on the page, where we finally append and call the functions at the end.

The CSS is then used to properly space and contain the information (particularly between one another), add a color gradient behind the cards, and the white color to create the 'card' (which was just the information from the function).
